# Web Research Refresh Pack

## What This Pack Demonstrates

This Tandem Workspace Pack showcases several key capabilities:

1. **Stale Document Analysis**: Identifying outdated information in existing briefs
2. **Web Research Workflow**: Systematic verification of facts using authoritative sources
3. **Evidence Documentation**: Creating auditable logs of research sources and findings
4. **Source Verification**: Using webfetch to verify claims against official documentation
5. **Report Generation**: Producing polished, cited HTML reports with inline citations

## Pack Details

- **Estimated Completion Time**: 15-20 minutes
- **Complexity Level**: Beginner-Intermediate
- **Outputs Produced**:
  - Evidence log (sources.jsonl or evidence.md)
  - Updated facts sheet (updated_facts.md)
  - Web Research Report (HTML)

## Use Cases

- Refreshing outdated product documentation
- Fact-checking and verification workflows
- Demonstrating research methodology
- Creating auditable research reports
- Training on source verification

## Prerequisites

- Familiarity with fact-checking concepts
- Basic understanding of web research methodology
- Access to Tandem workspace environment

## Safety & Licensing

All content generated by this pack is:

- **Original**: Synthesized from verified sources, no third-party text
- **Safe**: No personal data, secrets, or copyrighted material
- **Usable**: Clear license for any purpose (public domain)

## Technical Notes

### Input Files

When Tandem processes this pack, files are read in this priority:

1. inputs/stale_brief.md (contains outdated claims)
2. inputs/verification_questions.md (research requirements)

### Output Naming Convention

Generated files follow this pattern:

- `outputs/evidence.md` - Source verification log
- `outputs/updated_facts.md` - Corrected facts sheet
- `outputs/web_research_report_YYYYMMDD.html` - Final report

### Citation Format

The final report uses inline numeric citations:

- Inline: [1], [2], [3]
- Sources section maps numbers to full citations with URLs and access dates

### Evidence Log Format

Evidence is logged with:

- Claim being verified
- Source URL
- Access date
- Finding (confirmed/contradicted/insufficient evidence)
