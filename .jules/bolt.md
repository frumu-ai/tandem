## 2024-06-25 - Avoid multiple useMemo mapped passes on large streams
**Learning:** In a codebase that streams thousands of logs (like `src/components/logs/LogsDrawer.tsx`), performing three separate `lines.map().filter()` inside separate `useMemo` hooks is incredibly wasteful. It creates 6 intermediate array allocations matching the size of the stream, every time the `lines` state updates, causing huge memory spikes and GC churn.
**Action:** Combine multiple array-wide extractors over the same dataset into a single `useMemo` block with one manual `for` loop, parsing matching criteria into `Set` structures to maintain the exact same functionality while dramatically slashing O(N) allocation and compute overhead.
## 2025-02-15 - React useMemo Array Filtering Optimization
**Learning:** Found a common performance anti-pattern in React components where an array is filtered multiple times inside separate `useMemo` blocks to derive independent metrics (e.g., getting counts of "running", "completed", "pending", and "failed" tool calls). This incurs an O(n) penalty and an intermediate array allocation for *each* filter call, putting unnecessary pressure on the GC on every render where the dependencies change.
**Action:** When deriving multiple values from a single array in React, combine them into a single `useMemo` block that uses a single-pass `for` loop, eliminating intermediate array creations and significantly reducing traversal overhead.
